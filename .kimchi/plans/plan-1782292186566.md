## Decision Log
| Decision | Choice | Rationale | Rejected Alternatives |
|---|---|---|---|
| Frontend framework | Vue 3 + Bootstrap 5 (per spec) | User explicitly chose per spec | Blade+Tailwind (faster but diverges); Hybrid (complex) |
| Database (dev) | SQLite | User chose for zero setup | MySQL (requires server) |
| Auth | Sanctum SPA mode | Standard for Vue SPA on Laravel | Passport (overkill), session-only (won't work for SPA) |
| API style | REST with /api prefix, JSON resources | Standard Laravel pattern | GraphQL (overkill), RPC-style (non-idiomatic) |
| Roles | Enum on users table | Simpler than separate role/permission tables for 6 roles | Spatie Permissions (heavy for 6 fixed roles) |
| Testing | PHPUnit feature tests + factory seeders | Laravel-native, works without extra deps | Pest (extra dep), no tests (risky) |

## Risks
- **Frontend setup may need iteration** — Vue 3 + Vite + Laravel mix requires careful plugin ordering. Mitigation: use Laravel's official `@vite` directives, follow Laravel 13 + Vue 3 community pattern.
- **Batch/FEFO logic is core** — incorrect FEFO will silently break sales later. Mitigation: dedicated service class with comprehensive tests, including edge cases (expired batches, insufficient stock).
- **Multi-unit conversions** — wrong conversion factor causes inventory drift. Mitigation: dedicated service class + tests in both directions.
- **Slice 3 is large** — 6 migrations + 6 models + 5 controllers + 4 services + 7 Vue pages. Mitigation: delegate in 2 sub-batches (backend, then frontend).
